Extending bindDom() with custom reactive attributes — new in v2.0.0-beta.1

data-show — Conditional Visibility

Inverse of built-in data-hidden. Shows element when value is truthy.

👋 This message is visible because showMessage is true.
showMessage:
View Code 
import { show } from 'lume-js/handlers';

const store = state({ showMessage: true });
bindDom(root, store, { handlers: [show] });

// HTML: <div data-show="showMessage">Visible!</div>

classToggle() — Dynamic CSS Classes

Toggle individual CSS classes based on state. Each class gets its own data-class-{name} attribute.

Styled Box

Classes update reactively based on state

active: highlight: selected:
View Code 
import { classToggle } from 'lume-js/handlers';

const store = state({ isActive: false, isHighlighted: false });
bindDom(root, store, {
  handlers: [classToggle('active', 'highlight', 'selected')]
});

// HTML: <div data-class-active="isActive"
//            data-class-highlight="isHighlighted">

boolAttr() — Boolean Attributes

Toggle any HTML boolean attribute: readonly, open, contenteditable, etc.

readonly:
View Code 
import { boolAttr } from 'lume-js/handlers';

const store = state({ isReadonly: false });
bindDom(root, store, { handlers: [boolAttr('readonly')] });

// HTML: <input data-readonly="isReadonly" />

ariaAttr() — ARIA Attributes

Beyond built-in aria-expanded / aria-hidden. Add any ARIA state attribute.

aria-pressed: aria-selected:
View Code 
import { ariaAttr } from 'lume-js/handlers';

const store = state({ isPressed: false });
bindDom(root, store, {
  handlers: [ariaAttr('pressed'), ariaAttr('selected')]
});

// HTML: <button data-aria-pressed="isPressed"
//               role="button">Toggle</button>

stringAttr() — Dynamic String Attributes

Set href, src, title, or any attribute as a reactive string.

Dynamic Link →
View Code 
import { stringAttr } from 'lume-js/handlers';

const store = state({ linkUrl: 'https://github.com', linkTitle: 'GitHub' });
bindDom(root, store, {
  handlers: [stringAttr('href'), stringAttr('title')]
});

// HTML: <a data-href="linkUrl" data-title="linkTitle">Link</a>

Presets — Ready-Made Handler Bundles

Use formHandlers and a11yHandlers for common patterns without importing individual handlers.

View Code 
import { formHandlers, a11yHandlers } from 'lume-js/handlers';

// formHandlers includes: boolAttr('readonly')
// a11yHandlers includes: ariaAttr('pressed'), ariaAttr('selected'), ariaAttr('disabled')

bindDom(root, store, {
  handlers: [formHandlers, a11yHandlers]
  // Arrays are auto-flattened, so presets compose naturally
});

Custom Handlers — Build Your Own

Any object with attr and apply(el, val) works as a handler.

Hover me!

I have a dynamic tooltip (native title attribute)

View Code 
// Define a custom handler — just a plain object
const tooltip = {
  attr: 'data-tooltip',
  apply(el, val) { el.title = val ?? ''; }
};

const store = state({ tooltipText: 'Hello from Lume!' });
bindDom(root, store, { handlers: [tooltip] });

// HTML: <div data-tooltip="tooltipText">Hover me</div>
