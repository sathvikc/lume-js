repeat(): animated keyed grid

Every tile below is one keyed item in a single repeat() call. Reordering the array moves the same DOM nodes (no teardown, no re-creation), which is what lets a FLIP transition glide each tile from its old position to its new one. The library does the keyed reconciliation; the animation is ~20 lines of standard JavaScript on top.

How the animation works

repeat() keys each element by tile id, so a reorder is applied with insertBefore on the existing nodes. That identity is what FLIP needs: measure every tile's position, change the array, measure again after the reconcile (still before the browser paints), then apply the inverted delta as a transform and let it transition back to zero. 

const first = measure(board);        // old positions
store.tiles = shuffle(store.tiles);  // repeat() reorders the same nodes
await Promise.resolve();             // reconcile runs in Lume's flush
const moved = invert(board, first);  // transform: translate(old - new)
play(moved);                         // transition back to transform: none

Transforms animate on the compositor, so a thousand tiles glide at once without layout work per frame. New tiles scale in; the count slider, add, and remove reconcile through the same keyed path. The whole page runs on lume-js + the repeat addon, no build step, valid HTML.
